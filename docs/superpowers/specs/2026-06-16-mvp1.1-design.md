# MVP1.1 RAG 工作流升级设计

## 目标

在现有 `qa_rag_vehicle` 基础上升级检索链路：拆分改写、双路检索、串行核对，生成侧保持 `qwen3.6-plus` 不变。

## 流水线

```
q → [串行] keyword 提取 (qwen2.5:7b)
  → [串行] rewritten_q 改写 (qwen2.5:7b)
  → keyword Hybrid top2 + rewritten_q Hybrid top5
  → 去重合并 (5~7 条) + section_path boost
  → [串行 for] qwen2.5:7b 逐条核对 → 0~7 条
  → qwen3.6-plus 生成
```

## 约束

- 所有 Ollama 调用串行，不并发
- 不改 `ingest/`、不 rebuild 索引
- 标签方案 B：仅用现有 `section_path` 做检索 boost（×1.1），可选关闭
- 生成模型不变

## 模块改动

| 文件 | 改动 |
|------|------|
| `prompts/template.py` | 新增 `KEYWORD_EXTRACT_TEMPLATE`、`CHUNK_VERIFY_TEMPLATE` |
| `retrieve/query_rewrite.py` | `RewriteResult`、两次串行 Ollama |
| `retrieve/pipeline.py` | 双路检索、合并、接入 verifier |
| `retrieve/verifier.py` | 新建，串行核对 |
| `retrieve/hybrid.py` | `apply_section_boost()` |
| `config.yaml` | `keyword_top_k`、`rewritten_top_k`、`verification` |
| `main.py` | 日志展示 keyword / 核对结果 |

## 配置

```yaml
retrieval:
  keyword_top_k: 2
  rewritten_top_k: 5
  section_boost: 1.1

verification:
  enabled: true
  model: "qwen2.5:7b"
  temperature: 0.0
```
